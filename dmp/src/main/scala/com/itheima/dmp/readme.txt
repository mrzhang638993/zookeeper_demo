http的底层对应的是tcp协议的。
tcp：协议。包括报文格式和交互方式等一整套的操作的。
tcp/ip协议：tcp规定实现了数据从一端传递到另外的一端的，tcp可以保证两台计算机之间可以通讯的。
客户端和服务端的交互对应的是数据的交互的。程序的本质是数据加上算法的。程序的本质是使用算法操作数据的。

http中怎么体现出来tcp的操作的？
http底层操作的时候，首先使用tcp创建连接，发送数据的。发送数据完成之后，关闭tcp连接的。
socket是对于tcp协议的封装操作的，这样的话，我们可以使用更加简单的方式实现tcp协议之间的数据的传递操作的。
只有tcp协议是可以保证找到网络上的双方的，可以保证数据之间的交互的。http请求的话是不需要找到对方的。http可以通过底层的操作实现更多的操作实现的。
客户端和服务端的通讯操作的话，对应的都是可以使用http协议的。
HTTPClient很少使用了，不推荐使用httpClient进行http操作的。推荐使用okHttp进行http请求的操作的。
http2.0的多路复用机制的。okhttpclient的2.0对应的可以实现httpclient的操作实现的。
