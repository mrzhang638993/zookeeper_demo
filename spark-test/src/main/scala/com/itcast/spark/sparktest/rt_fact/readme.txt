这里面可以实时的统计出来某个key的实时的总量的
但是还是存在一个区别:那就是无法得到当天的实时的统计总量的
计算的操作是可以如下的：redis的key中包含了当天的数据信息的。
redis的key中需要包含当天的时间信息的。这样的话，可以完美的完成任务的。
或者是mysql中存储了这样的数据的,但是mysql计算的话,可能计算太慢了。
可以采用一个定时任务的方式来消费redis中的数据保存到mysql中进行持久化的。
同时还可以根据redis的key来同步数据到mysql中来完成持久化操作的。
