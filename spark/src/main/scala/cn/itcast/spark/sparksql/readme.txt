1.命令式编程和声明式编程(sql)
命令式编程：对应的是这种关注命令的每一步的操作实现的，需要理解每一步操作的
声明式编程：关注结果的。


hive:对应的是一个mapreduce的客户端的，执行的时间特别的长的。
shark:使用spark的rdd代替了hive的sql计划的。shark对应的是基于进程的。
spark sql:对应的是基于线程的。支持声明式的api和命令式的api操作的。底层是基于spark的rdd操作的
使用的最多的是spark sql的，而不是rdd的。


三种数据集：
1.结构化数据；mysql的数据表，存在非常明确的schema约束的。对于字段类型都有约束的。字段也有约束。
2.半结构化数据;有字段的约束，有字段类型的，和结构化的数据的区别在于没有一个严格的schema约束的。可以修改schema约束的
3.非结构化数据:没有固定的schema,没有固定的

# spark的相关的使用范围和数据
Spark 的 RDD 主要用于处理非结构化数据和半结构化数据
SparkSQL 主要用于处理 结构化数据
spark sql是基于spark 的rdd的，但是整体的运行速度是相较而言速度更加的快的。在提供更多的功能的时候，提供更加快的速度的。
